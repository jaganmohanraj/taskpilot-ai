# Operating Model

## Standard runbook

1. Capture the request.
2. Create a project.
3. Set objective and acceptance criteria.
4. Break work into steps.
5. Keep status current after each meaningful action.
6. Log decisions, blockers, assumptions, and evidence.
7. Run completion audit.
8. Close only if verified.

## Anti-drift rules

- do not skip state updates
- do not close with open tasks
- do not ignore blockers
- do not mark assumptions as facts until verified
